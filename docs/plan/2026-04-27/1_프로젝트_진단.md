## 2. 의도와 실제의 정합성 진단

### 2-1. 정합 (의도대로 잘 가는 부분) ✅

1. **"미니홈피"라는 데모의 선택이 매우 영리**
   - 싸이월드 미니홈피는 30~40대 한국인의 **공통 노스탤지어**다. 이 톤이 살아 있는 시점은 2026년 1차 강의 페르소나(20대 후반~40대 직장인)에 정확히 꽂힌다.
   - "2시간 만에 미니홈피 만들기" 같은 카피는 **인스타 릴스/유튜브 숏츠로 뽑기 좋은 시각적 결과물**(별 파티클, 방문자카운터)을 자연스럽게 포함한다.
   - 즉 미니홈피는 **결과물 = 마케팅 콘텐츠 = 다음 회차 모집 자석** 의 삼박자가 맞는 매우 좋은 선택.

2. **CLAUDE.md의 모바일 우선 룰이 잘 살아 있음**
   - 1-5-1 (모바일 카드 그리드 2열), 1-5-2 (아코디언) 같은 룰이 명문화돼 있고 코드에서도 `grid-cols-2 sm:grid-cols-2 lg:grid-cols-3` 패턴이 일관됨.
   - 인스타·당근에서 들어온 모바일 트래픽이 거의 100%인 시장 특성에 맞음.

3. **"확인 없이 바로 진행"의 작업 룰 + 작업 로그 디시플린**
   - `docs/task/YYYY-MM-DD/HHMM_*.md` 패턴으로 의사결정 이유가 자산화되고 있음.
   - 1인 운영자가 6개월 후 자기 결정을 추적할 수 있는 유일한 방법. **이 디시플린이 살아 있어야 탈주 후에도 안 무너진다.**

4. **단계 진화 정책의 일관성**
   - 결제(계좌 → PG), 채팅(카톡 오픈채팅 → 자체 Socket.io), 인프라(IP → 도메인+SSL) 모두 "필요한 트리거가 채워지면 도입" 으로 통일.
   - 1인 운영자가 가장 많이 망하는 케이스는 "있어 보이는 기능 다 박다가 정작 본업(강의)을 못 굴리는 것". 이걸 처음부터 방지하는 구조.

### 2-2. 어긋나기 시작한 부분 ⚠️

1. **Next.js → React (Vite + React Router) 전환의 비용**
   - 1차 기획안에서는 Next.js 14 App Router로 명시됐으나 코드는 Vite + React Router로 가고 있음.
   - 잃는 것: SSR/SSG, generateStaticParams, 자동 SEO 메타. **블로그 SEO 유입이 사업의 트래픽 자석 중 하나라는 1차 기획안의 가설을 그대로 두면 곧 충돌이 온다.**
   - 얻는 것: 빌드 단순함, 학습 곡선 낮음. 정적 사이트라면 Vite도 충분.
   - **권고**: 의식적으로 결정한 게 아니라면 결정 문서를 한 번 남길 것. SEO 트래픽을 포기할지, 아니면 학습 라이브러리/블로그만 별도 Next.js 서브도메인으로 분리할지의 분기.

2. **어그리게이션 대시보드의 "외부 모임 시드" 가 안 채워지는 중**
   - 1차 데모 기획에서 "외부 시드 12~15건"을 손으로 채우라고 했지만, `events.json` 시드 충실도가 미확인.
   - **이게 비어 있으면 사이트의 가장 강한 차별점("한국 바이브코딩 모임 메타 검색")이 무력화** 된다. 미니홈피 강의 콘텐츠는 강해지지만, 사이트 자체의 트래픽 자석은 약해지는 비대칭 발생.
   - 권고: 미니홈피 코드 작업 1시간당 외부 모임 시드 30분 매칭으로 균형 맞출 것.

3. **"5~10명 인터뷰" 검증 단계가 건너뛰어진 것 같음**
   - 1차 데모 기획의 진짜 산출물은 인터뷰 4가지 가설의 답이었음.
   - 지금은 인터뷰 없이 미니홈피 콘텐츠 → 4주차 커리큘럼으로 빨리 진도가 나가고 있음.
   - **이게 위험한 이유**: 콘텐츠는 만들면 만들수록 "이거 쓸데 있나" 의심을 못 함(매몰비용). 5~10명 피드백 없이 4주 콘텐츠를 다 만들면 "어그리게이션 대시보드의 차별성"보다 "강의 콘텐츠"에 정체성이 쏠려서 결국 **클래스101/탈잉이 이미 잘하는 영역에서 1인 운영자로 경쟁** 하게 됨.

4. **사업자 등록·통신판매업 신고가 결제 의사결정과 분리되지 않음**
   - 1차 기획안에서 PG 도입 트리거는 명확(월 매출 100만/신청자 40명/노쇼율 15~20%). 그러나 "그 시점에 사업자 등록을 미리 해둘지" 의 선행 작업이 빠짐.
   - 사업자 등록 → 통신판매업 신고 → PG 심사가 1~2주 걸린다. **트리거가 채워진 그달부터 사업자 등록을 시작하면 다음달 매출의 환불·영수증이 깨진다.** 트리거 직전 1개월 시점에 미리 등록을 시작해야 함.
   - 권고: "월 매출 60만" 또는 "월 신청자 25명" 시점에 사업자 등록 사전 작업 시작.

5. **"P6ix 탈주" 라는 본인 목표가 사이트에 명시되지 않음**
   - 모든 의사결정의 시간 압박을 결정짓는 변수인데, 코드/문서 어디에도 이 변수가 안 보임.
   - 1인 운영자가 사이드 프로젝트로 시작한 것을 본업으로 전환하는 구간은 **목표가 명시되지 않으면 영원히 사이드로만 머무른다.**
   - 권고: 비공개 메모(`docs/personal/`)에 "탈주 트리거" 6개월 단위 마일스톤을 명시. 외부 노출은 안 하더라도 본인이 매월 자기 진척을 그 척도로 측정할 수 있어야 함. (이 리포트 5번에서 상세)

### 2-3. 잠재 리스크 (지금은 문제 아니지만 6개월 내 터질 수 있음) 🔥

1. **콘텐츠 제작 → 강의 운영 → 마케팅의 1인 분업 한계**
   - 강의 1회 = 콘텐츠 준비 4시간 + 모객 2시간 + 강의 2시간 + 후기 정리 1시간 + 다음 회차 마케팅 3시간 = 약 12시간/회.
   - 회당 6명 × ₩25,000 = ₩150,000 매출. 시급 환산 ₩12,500. 본업 시급 대비 분명히 낮음.
   - 이 비효율을 못 깨면 회사 탈주가 불가능. **"같은 콘텐츠를 N회 재사용"** 의 자동화가 핵심 (영상강의 / 전자책 / 멤버십).

2. **수원 거점의 시장 한계**
   - 수원-서울 라인은 인구 풀이 크지만, 정기 주말 모임에 매주 신규 6명이 들어오려면 광역 마케팅이 필요.
   - 광교/수원역에서 주 1회 정원 6명이 4주 연속 차려면 누적 24명. 수원 단독 페르소나로는 빠듯.
   - 권고: 강남/사당/판교 등 서울 남부와의 듀얼 거점을 고려. 또는 회차당 격주.

3. **AI 도구의 빠른 변화**
   - Cursor/Claude/ChatGPT의 UI는 6개월마다 변함. 미니홈피 강의 자료가 1년 후엔 6~70% 갱신 필요.
   - 권고: 콘텐츠를 "도구 사용법" 보다 "AI에게 잘 설명하는 사고법" 으로 잡는 1차 기획안의 방향이 정답. **이 방향성을 미니홈피 강의 교안에 명시적으로 반영해야 콘텐츠 수명이 길어짐.**

4. **번아웃**
   - 본업 + 사이드 강의를 6개월 이상 동시 진행하면 번아웃 거의 확실.
   - 권고: **1주 1회 강의로 한정**. 회사 일이 바쁜 주는 격주로 줄이거나 외부 모임으로 트래픽 우회.

## 3. 사용자(본인)와 이 사업의 적합성 평가

> 이건 "잘 맞는다/안 맞는다" 가 아니라 "어떤 부분이 잘 맞고, 어떤 부분이 보강 필요한가" 의 진단.

### 3-1. 강점 (확실히 본인에게 맞는 부분)
- **코더 본인이 운영자** = 콘텐츠 발화량/진정성 최고. 클래스101 강사처럼 외주가 아니라 본인이 직접 만든 것을 보여주는 신뢰감.
- **"바이브 세션" 톤 (거창하지 않고 친근)** = 본인의 1차 기획안 톤 키워드와 맞음. 토스/당근/Airbnb 레퍼런스 일관.
- **반복적 디시플린** = `docs/task` 로그, `agents/` 분류, CLAUDE.md 룰 명문화 — 1인 운영자에게 가장 중요한 자산.
- **단계 진화 사고** = 결제/채팅/인프라 모두 "필요할 때 도입" 의 단계화. 과잉 투자 안 함.

### 3-2. 보강 필요 (보이는 약점 또는 미지)
- **마케팅(특히 인스타·릴스 콘텐츠 제작)** 의 디시플린이 코드만큼 강한지 미지수. 1인 운영의 매출은 결국 마케팅 자체.
- **타인 앞에서 강의하는 멘탈/체력** 이 미지수. 첫 1~2회는 6명 앞에서도 어색할 수 있음.
- **재정 안전망** (탈주 후 6~12개월 무수입 버틸 수 있는지) 미지수.

### 3-3. 본인 페르소나의 강점을 살리는 방향
- "회사 다니는 직장인이 만든 모임" = 페르소나 신뢰감.
- **"6개월 안에 탈주를 검증하기 위한 실험"** 으로 사업 자체를 프레이밍하면, 콘텐츠 자체에 진정성·시급성이 묻어남. 이건 마케팅 자산이기도 함 (인스타 스토리 시리즈로 공개 가능).

## 4. 다음 30일 권고 (이번 분기점에서)

### 4-1. 안 미루면 좋은 것
1. **미니홈피 4주 강의의 첫 회차를 5월 안에 한 번 굴려라**. 사이트 미완성 OK. 소모임 앱 + 구글 폼 + 카톡 충분.
2. **외부 모임 시드 12~15건을 events.json에 손으로 채워라**. 이게 어그리게이션 대시보드의 차별점이다.
3. **"내가 이 사이트로 탈주를 노리고 있다" 를 비공개 노트에 한 번 명시하라**. 의사결정의 시간 변수가 없는 상태에서는 모든 결정이 미뤄진다.

### 4-2. 미뤄도 되는 것
- 사업자 등록 / 통신판매 신고 / PG 도입 (Phase 2 트리거까지)
- 자체 채팅 / 멤버십 정기결제 / 영상 호스팅 (Phase 3~4)
- 도메인 / SSL (도메인 결정되면 그때)

### 4-3. 가장 위험한 것
- **인터뷰 없이 4주 강의 콘텐츠 제작에 80시간을 쏟는 것**. 5명에게만 데모 보여주고 4가지 가설에 답을 들으면 80시간 중 30시간을 절약할 수 있음.
