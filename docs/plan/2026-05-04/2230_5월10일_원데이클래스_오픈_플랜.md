## 0. 한 줄 요약

5/10 일요일 첫 원데이 클래스를 **시간단위 예약 스터디룸 + 카톡/소모임 신청 + 계좌이체** 흐름으로 연다.
사이트는 **안내·교안·신뢰 페이지**만 담당하고, **DB·신청 폼은 만들지 않는다 (Phase 0 유지).**

---

## 1. 장소 + 노쇼 리스크

### 결정
- **장소 정책**: 시간단위 예약 가능 + 24h 전 무료 취소 가능한 스터디룸으로 잡는다.
  플랫폼 후보: 스페이스클라우드 / 네이버 예약. 1.5~3만 원선, 보증금 결제 없는 곳 다수.
- **지역 후보**: 강남 / 홍대 / 성수 (운영자 동선 기준).
- **첫 회는 운영자 본인이 스터디룸비 부담**한다. 한 자릿수만 원선 손해는 1회 광고비로 본다.

### 리스크 보호장치 (사이트 + 소모임 양쪽에 명시)
> "최소 인원 3명 미만 시 다음 회차로 연기합니다. 입금하신 분은 100% 환불해드립니다."

이 한 줄을 사이트 클래스 카드 + 소모임 모집 글에 동일하게 박는다.
운영자 본인의 멘탈 보호 + 신청자 안심 둘 다 가능.

### 미정
- 실제 잡을 장소 1곳 선택 (또는 site.json 더미 "강남역 PLAY 스터디룸" 그대로 박고 추후 교체)

### 필수 필드
| 필드 | 예시 |
|------|------|
| 제목 | "Claude로 첫 웹사이트 만들기 — 1주차" |
| 한 줄 설명 | 비전공자도 두 시간이면 첫 화면이 뜬다 |
| 일시 | 2026-05-10 (일) 14:00–17:00 |
| 장소명 | 강남역 PLAY 스터디룸 (예시) |
| 주소 + 출구 안내 | 강남역 11번 출구 도보 5분 |
| 지도 마커 | 기존 LeafletSingleMap 재사용 |
| 정원 / 현재 인원 | "정원 6명 / 현재 N명" — **수동 갱신** |
| 가격 | 미정 (1만 / 2만 / 3만 후보) |
| 계좌번호 + 예금주 | 운영자 계좌 |
| 입금자명 규칙 | "이름+휴대폰뒷4자리" |
| 교안 링크 | `/guide/oneday/week-1` 등 기존 라우트로 직링크 |
| 신청 CTA | 카톡 오픈채팅 딥링크 |

### 같이 박아두면 좋은 거
- **준비물 체크리스트**: 노트북 / Claude Code 설치 / VSCode / 인터넷 / 충전기
- **사전 질문 폼**: 구글폼 한 줄 ("코딩 경험 / 만들고 싶은 것")
- **환불 정책**: 이미 `site.json.refundPolicy` 있음 → 재사용
- **인원 미달 시 연기 정책**: 위 § 1 의 한 줄
- **강의 후기 자리**: 첫 회는 비워두고 1회차 끝나면 채움

## 3. DB 도입 — 보류 (Phase 0 유지)

### 결정
**소모임/카톡 신청 + 계좌이체** 흐름을 유지한다. 사이트에 신청 폼·결제·DB 박지 않는다.

### 이유
1. **CLAUDE.md § 3 결제 정책**: "Phase 2 진입 시점까지 PG 자동 결제 흐름은 사이트에 박지 않는다."
2. **시간 부족**: 5/10까지 6일. 신청 폼 + 백엔드 API + 입금 확인 로직 짜고 디버깅하면 본업·교안 보강 시간이 깎인다.
3. **데이터 0**: 첫 회는 신청자 패턴 자체가 없어서 폼 설계가 추측이 됨. 1~3회 굴려보고 "사람들이 실제로 뭘 묻는지" 본 다음에 폼을 짜야 정합성 있음.
4. **역할 분리**: 사이트 = 안내·교안·신뢰 / 소모임·카톡 = 신청·결제·CS. 깔끔하고 본인 운영 부담 적음.

### DB 도입 트리거 (이때 가서 만든다)
- 회당 신청자 5명 이상이 2~3회 연속
- 카톡으로 신청 받기 번거로워지는 시점
- 노쇼율이 신경 쓰이는 시점
