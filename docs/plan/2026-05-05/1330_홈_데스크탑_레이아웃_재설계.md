# 홈 데스크탑 레이아웃 재설계 — 자료 적을 때 풍성하게

작성일: 2026-05-05
범위: **데스크탑(lg+ 1024px+) 만**. 모바일·태블릿(< lg) 은 건드리지 않음.
배경: 강의 1개 + 4주차 시리즈만 있어 데스크탑 풀폭에서 콘텐츠가 비어 보임. 인프런처럼 카테고리 풍부하지 않은 상태에서 균형 잡힌 그림이 필요.

---

## 1. 현재 비대칭의 원인 진단

```
┌─────────────────────────────────────────────────────────────┐  ← 데스크탑 max-w-7xl (1280px)
│                          헤더                                 │
├─────────────────────────────────────────────────────────────┤
│ 운영자 명함  │  수업 전 준비   │  운영자 추천 (큰 미니홈피)    │  ← 3등분 (어색)
│ (텍스트만)   │  (작은 썸네일)  │  (큰 썸네일)                  │
└──────────────┴────────────────┴───────────────────────────────┘
┌─────────────────────────────────────┬───────────────────────┐
│ 다음 강의 미리보기 (1주차) +        │                        │
│ 2주차 / 3주차 / 4주차 (가로 4장)    │      지도              │  ← 2등분
│                                      │                        │
│ + 새 강의 카드 (admin)              │                        │
└─────────────────────────────────────┴───────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│ 처음이세요? 5스텝                                              │
└─────────────────────────────────────────────────────────────┘
```

### 문제 3가지

1. **상단 3등분** — 시각 무게가 안 맞음. 중간 카드가 작은 썸네일이라 좌우와 균형 X
2. **상단 3 → 중단 2 → 하단 1 단의 비율 충돌** — 한 페이지 안에 컬럼 수가 매번 바뀌어 시선이 갈팡질팡
3. **중단 좌측에 4주차 시리즈가 4장 가로로 박혀** — 옆에 지도가 있는데 또 가로 4장이라 빽빽

근본 원인: **콘텐츠 양과 레이아웃 폭이 안 맞음.** 데스크탑 1280px 에 강의 1개 + 시리즈 미리보기 = 풀폭 채우려고 억지 분할.

## 2. 데스크탑 레이아웃 옵션 3가지

### 옵션 A — 메인 + 우측 Sticky 사이드 (Airbnb 식)

> 변경 최소. 현재 구조에서 우측 사이드만 sticky 로 분리.

```
┌─────────────────────────────────────────────────────────────┐
│                          헤더                                 │
└─────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────┬──────────────────────┐
│ HERO (지금 이 동네에서)               │                      │
│ ─────────────────────────────         │   운영자 명함         │
│ 운영자 명함 (한 줄)                   │   (sticky 카드)       │
│                                       │                      │
│ ─────────────────────────────         │   ─────────────      │
│ ★ 운영자 추천 (5/10)                 │   문의/카톡 CTA      │
│   큰 강의 카드 1장                    │                      │
│                                       │   ─────────────      │
│ ─────────────────────────────         │   처음이세요? 5스텝  │
│ 4주차 시리즈 미리보기                 │                      │
│ ┌────┬────┬────┬────┐                 │                      │
│ │1주차│2주차│3주차│4주차│             │                      │
│ └────┴────┴────┴────┘                 │                      │
│                                       │                      │
│ ─────────────────────────────         │                      │
│ 지도 + 카드 매칭                      │                      │
│ (지도와 외부/자체 마커)               │                      │
│                                       │                      │
│ ─────────────────────────────         │                      │
│ 캘린더 (전체)                         │                      │
└──────────────────────────────────────┴──────────────────────┘
   메인 콘텐츠 ~720px                    사이드 ~280px sticky
```

**장점**: 시선이 좌→우로 한 번만. 사이드 sticky 라 항상 신청 진입점 보임.
**단점**: 자료 더 늘면 사이드가 좁아 답답.

---

### 옵션 B — 세로 스토리텔링 + 너비 제한 (Toss / Notion 식) ⭐ 추천

> 데스크탑 max-width 를 좁혀(`lg:max-w-3xl` ~ `lg:max-w-4xl`) 자료 적을 때 풍성해 보이게.

```
┌─────────────────────────────────────────────────────────────┐
│                          헤더                                 │
└─────────────────────────────────────────────────────────────┘

           ┌─────────────────────────────────────┐
           │ HERO                                 │
           │ "지금 이 동네에서, 뭐 하지"          │
           │ 한 줄 설명                           │
           └─────────────────────────────────────┘

           ┌─────────────────────────────────────┐
           │ 운영자 명함 (가로 풀폭)              │
           │ 사진 + 이름/한줄/연락처 인라인        │
           └─────────────────────────────────────┘

           ┌─────────────────────────────────────┐
           │ ★ 다음 강의 — 5/10 (큰 카드)        │
           │                                       │
           │ ┌──────────┐                          │
           │ │ 썸네일   │   제목                   │
           │ │ 16:10    │   D-5 · 25,000원         │
           │ └──────────┘   "신청하기 →"           │
           │                                       │
           └─────────────────────────────────────┘

           ┌─────────┬─────────────────────────┐
           │ 지도     │  4주차 시리즈 카탈로그   │
           │ 작게    │  (1주차/2주차/3주차/4주차) │
           │ 한 카드  │  세로 4장 작게           │
           └─────────┴─────────────────────────┘

           ┌─────────────────────────────────────┐
           │ 캘린더 (선택적, 토글 또는 모달)      │
           └─────────────────────────────────────┘

           ┌─────────────────────────────────────┐
           │ 처음이세요? — 5스텝 (한 줄씩)        │
           │ + 카톡 CTA                            │
           └─────────────────────────────────────┘
```

**장점**:
- 한 줄로 흐르는 스토리 → 비어 보임 X
- 카드 키워서 시각 무게 살림
- 자료 늘어날 때 자연스럽게 카드만 추가 (구조 무변화)

**단점**: 데스크탑이 와이드 모니터일 때 좌우 빈 공간이 큼 → 다만 그게 "여백" 으로 보이게 배경 처리 가능.

### 옵션 C — 큰 Hero 배너 + 그리드 (인프런 / 배달의민족 식)

> 메인 강의 1개를 풀폭 배너로. 그 아래는 시리즈/지도 카드 그리드.

```
┌─────────────────────────────────────────────────────────────┐
│                          헤더                                 │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│  HERO BANNER ─ 풀폭 큰 강의 카드 (5/10)                       │
│                                                                │
│  ┌──────────────┐    "AI랑 만드는 내 미니홈피"                │
│  │              │     코딩 처음 환영 · 2시간 입문              │
│  │  미니홈피    │     5/10 (일) 14:00                          │
│  │  라이브 데모 │     커피에스터디 서초 · 4명 · 25,000원       │
│  │              │                                                │
│  └──────────────┘     ┌──────────────────────┐                │
│                        │   신청 페이지 →        │                │
│                        └──────────────────────┘                │
└─────────────────────────────────────────────────────────────┘
┌────────────────┬────────────────┬────────────────────────────┐
│ 1주차          │ 2주차          │  지도                       │
│ 미니홈피       │ 프로필+카운터  │  (강남/서초 중심)            │
│ ★ 진행중      │ 오픈 예정      │                              │
├────────────────┼────────────────┤  + 마커 클릭 → 카드 강조    │
│ 3주차          │ 4주차          │                              │
│ 카드+다이어리  │ 꾸미기+배포    │                              │
│ 오픈 예정      │ 오픈 예정      │                              │
└────────────────┴────────────────┴────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│ 처음이세요? — 5스텝 (가로 배치) + 카톡 CTA                    │
└─────────────────────────────────────────────────────────────┘
```

**장점**:
- 첫 진입 임팩트 큼. 5/10 강의가 즉시 눈에
- 인프런·배민 패턴이라 사용자 친숙
- 지도가 카드 그리드와 같은 단에 있어 균형

**단점**:
- 배너 하나에 시각 무게 다 몰림 → 다른 회차 추가 시 어색해질 수 있음
- 강의가 5개 이상 쌓이면 배너만 1개 보여주는 게 손해

## 3. 비교 표

| 기준 | A. 메인+사이드 | B. 스토리텔링 ⭐ | C. 큰 Hero |
|------|---------------|----------------|-----------|
| 현재 자료량(강의 1개) 적합도 | 보통 | **최상** | 좋음 |
| 자료 늘어날 때(10+ 강의) | 보통 | 좋음 | 나쁨 (배너 어색) |
| 모바일 영향 | 0 | 0 | 0 |
| 코드 작업량 | 중 (사이드 분리) | **소** (max-width 좁히고 섹션 정렬) | 중 (배너 컴포넌트 신규) |
| 운영자 본인 톤(친근·미니멀) | 보통 | **최상** | 보통 |
| 신청 진입점 가시성 | 사이드에 항상 보임 | 스토리 중간에 1번 | 배너에 1번 |

## 4. 추천 — 옵션 B (스토리텔링 + 너비 제한)

이유:
1. **현 자료량(강의 1개)에 자연스럽게 맞음** — 적은 자료가 빈 공간 없이 풍성해 보임
2. **운영자 톤("거창하지 않고 친근하고 비싸 보이지 않는") 부합** — Toss/Notion 식 단일 컬럼 = 차분
3. **자료 늘어나도 무수정** — 카드만 추가하면 됨, 구조 변경 0
4. **모바일 무영향** — `lg:max-w-3xl mx-auto` 하나만 바꾸면 됨. `< lg` 에서는 기존 그대로

### 구체 변경안 (옵션 B 채택 시)

```diff
HomeClient 의 최상위 wrapper:
- <div className="container-wide">
+ <div className="container-wide lg:max-w-3xl xl:max-w-4xl">

각 섹션 (운영자 명함 / 다음 강의 / 시리즈 / 지도+캘린더 / 5스텝):
  세로 stack, 사이 `mt-8 sm:mt-12` 로 충분한 여백

기존 좌우 분할 영역:
- 데스크탑 2-column 분리 제거
- 모바일 동작은 그대로 (모바일은 어차피 1열)

지도 + 시리즈는 좁은 폭에서 가로 2등분 가능:
- <div className="lg:grid lg:grid-cols-2 lg:gap-6">
```

추가 권장:
- HERO 옆에 작은 일러스트 또는 운영자 사진 작게 → 텍스트 단조로움 완화
- "다음 강의" 카드 자체를 키움 (썸네일 16:10 → 16:9 더 큰 비율)
- 카톡 CTA 는 페이지 하단 고정 + 모바일 sticky bottom (현재 모바일 OK 면 그대로)

## 5. 안 한 것 / 결정 보류

- **컬러·폰트·일러스트 추가** — 본 작업 범위 밖. 레이아웃 결정 후 별도
- **3D 일러스트나 영상 hero** — 본인 톤 안 맞음 (비싸 보임)
- **카드 호버 애니메이션 강화** — 모바일은 hover 없음 룰
- **외부 모임 검색 강화** — 외부 데이터 풍부해질 때 재검토

---

## 6. 결정 요청

OK 면 옵션 B 로 코드 진행:

1. `HomeClient.jsx` 최상위 컨테이너에 `lg:max-w-3xl xl:max-w-4xl` 추가
2. 데스크탑 2-column 분할 영역(메인 카드 + 지도) 제거 → 세로 stack 으로 통일
3. 지도 + 4주차 시리즈는 좁은 폭에서 `lg:grid-cols-2` 로 균형
4. 섹션 간 여백 `mt-8 sm:mt-12` 로 호흡
5. 운영자 명함 + 운영자 추천 카드 통합 검토 (시각 무게 균형)

다른 옵션(A 또는 C) 선호하면 알려줘. 또는 이 안에서 일부 요소만 빼거나 더하고 싶은 거 있으면 그것도.
