## 0. 정체성 한 줄 확정

> **강의를 안내하고 교안을 쌓아두는 사이트. 일정은 캘린더로 보여주고, 신청·운영은 카톡으로.**

빠진 것 / 남은 것 명확히:
- ❌ 모임 모집 플랫폼 X — 소모임/카톡이 함
- ❌ 모각작·외부 모임 카탈로그 X — 외부 사이트가 함
- ❌ 지식 글 블로그 X — 보류 (1차 강의 후 결정)
- ✅ 자체 강의 신청 진입점
- ✅ 교안(Curriculum) 본문 + N회차 매핑
- ✅ 캘린더로 자체 강의 일정 한눈에

## 1. 영역별 결정

| 영역 | 현재 | 결정 |
|------|------|------|
| **지도** | 카카오맵 풀폭 | **유지** — 다음 강의 카드 바로 아래 박음. 운영자 use case ("위치가 가까운지") 부합 |
| **캘린더** | 모달/풀폭 | **유지하되 접힘(collapsed) 기본** — 헤더 클릭으로 펼침/접힘 토글. 모바일 + 데스크탑 둘 다 |
| **외부 모임 (evt-101~111)** | isPublished:false | **데이터까지 삭제** — 사이트 정체성과 안 맞음, 부활 가능성 거의 0 |
| **자체/외부 토글 + 필터 패널** | 상단 필터 바 | **제거** — 외부 빠지면 의미 0 |
| **운영자 명함** | 상단 카드 | 유지 |
| **다음 강의 카드** | 풀폭 한 장 | **확대** — 메인 진입점, 큰 hero |
| **4주차 교안 카탈로그** | 별도 영역 | **메인 콘텐츠로 격상** |
| **바이브코딩 지식 글** | 없음 | **보류** (1차 강의 후 결정) |
| **처음이세요? 5스텝** | 하단 | 유지 (간소화) |

## 3. 자료 늘어날 때 진화 시나리오

| 자료량 | 변경되는 곳 | 변경 방식 |
|--------|------------|----------|
| **현재 (강의 1개)** | 위 구조 | 그대로 |
| **자체 강의 5개 (5/10·5/17·5/24·5/31·6/7 회차)** | "다음 강의" 영역 | 좌 큰 카드(가장 가까운) + 우 작은 카드 3장 또는 carousel. 캘린더에 ★ 5개 표시 |
| **2주차 교안 본문 채움** | "강의 교안" 카탈로그 | 2주차 카드만 "오픈예정" → "진행중" 갱신 |
| **시리즈 8주차+** | "강의 교안" 카탈로그 | `grid-cols-4` 유지 + "더 보기" 또는 페이지네이션 |
| **지식 글 영역 활성** | 캘린더 위/아래 새 섹션 | placeholder → 글 카드 grid |

골격은 안 바뀜. 섹션 안에서 카드 N장이 늘어날 뿐.

## 4. 영향 받는 코드 / 데이터

### 제거
- `front/src/components/HomeClient.jsx`
  - 자체/외부 토글 (`sourceFilter`) + 필터 패널 (`region`, `type`, `price`)
  - 캘린더 모달 (`calModal` state, 모달 div) → 인라인 Accordion 으로 이동
  - `recommended` 같은 외부 강의 추천 로직
- `front/src/data/events.json` + `front/public/data/events.json`
  - `evt-101` ~ `evt-111` 외부 이벤트 11건 **데이터 자체 삭제**
- 헤더의 "모임" 메뉴는 라벨 그대로 (강의 안내가 메인이라 의미 통함). 또는 "강의" 로 변경 검토

### 변경 / 위치 이동
- `KakaoMultiMap` — HomeClient 에 **유지** (제거 X). 다음 강의 카드 바로 아래에 박음. 자체 강의 마커만 노출 (외부 데이터 사라지면 자동으로 자체만 남음)
- `CalendarView` (HomeClient 내부 함수) — 모달 트리거 제거 → **`Accordion` 으로 감싼 인라인 섹션** (접힘 기본, 클릭으로 펼침). 모바일·데스크탑 동일
- `EventCard`, `ClassRegistration`, `ClassEditor`, `Week1`, `OnedayClassCurriculum`, `OperatorIntroCard`, admin 시스템 모두 그대로
- `/events/:id` 진입 → 그대로 (다음 강의 카드 또는 지도 마커 클릭 → 상세)

### 신규
- 큰 hero "다음 강의" 카드 컴포넌트 (현재 EventCard 와 다른 디자인 — 풀폭 좌-우 분할)
- `CurriculumGrid` 컴포넌트 (이미 사용자가 만들었음 — `pages/guide/index.jsx` 에서 import 중)

### CLAUDE.md 룰과의 관계
- `§ 1-5-2` "데스크탑(`lg:`)에서는 항상 펼친 상태 유지" 룰의 **예외**: 캘린더는 자료 적을 때 큰 빈 면적이라 데스크탑에서도 접힘 기본. 룰 자체는 일반 가이드, 캘린더가 그 예외 케이스. (룰 수정 X — 본 결정은 한 컴포넌트 한정 예외)
