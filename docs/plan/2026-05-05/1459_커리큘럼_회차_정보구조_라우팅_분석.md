# 커리큘럼 ↔ 회차 정보 구조 + 라우팅 분석 (2026-05-05 14:59)

## 0. 사용자 핵심 깨달음 (정리)

> "AI랑 만드는 내 미니홈피" 카드는 `/events/evt-week1-2026-05-10` 으로 가는 **실제 오픈된 강의(회차)**. "새 강의 추가" 영역은 `/guide/oneday/week1` 같은 **강의 커리큘럼**. 1주차는 실제 강의, 2~4주차는 준비중 커리큘럼. 그러면 메인의 그 영역에 "강의 커리큘럼 리스트 + 각 커리큘럼과 연결된 곧 열릴 회차" 가 같이 나와야 하는 거 아닌가? 별도 라우트도 만들어야 하나?

**정답**: 사용자 인지가 맞다. 단 **별도 라우트 신설은 불필요**. 그 역할은 이미 `/guide` 인덱스가 맡아야 함 (지금은 비어있을 뿐). 메인은 회차 중심, /guide 는 커리큘럼 중심으로 분리해야 함.

## 1. 현재 정보 구조 (혼란의 원인)

### 1-1. 두 도메인이 한 페이지에 섞여 있음

| 도메인 | 의미 | 파일 위치 | 라우트 |
|--------|------|----------|--------|
| **회차 (Class instance)** | 5/10 14시 강남, 25,000원, 잔여 4석 — 신청 가능한 단위 | `events.json` (회차 메타) | `/events/:id` |
| **커리큘럼 (Curriculum / Template)** | "1주차 — 미니홈피 만들기" 같은 교안 | `curriculums.json` + `pages/guide/oneday/Week*.jsx` | `/guide/oneday/week{N}` |

연결 고리: `event.curriculumId` ↔ `curriculum.id` (N:1 — 한 교안으로 N회 회차 열기)

### 1-2. 메인 페이지(`/`) 의 현재 모습

```
[히어로 + 필터]
[카드 3종: 운영자 / 사전가이드 / 운영자추천]   ← 운영자추천 = 회차 1건 강조
[카드 그리드 + 지도]                              ← 회차 카드들 (현재 0건)
```

문제:
- 회차 1건뿐이라 카드 그리드가 빈 채로 "조건에 맞는 모임이 없어요" 표시
- 사용자가 "어떤 강의들이 있고 다음 회차는 언제인지" 못 봄
- 진입자 입장에서 사이트가 한산해 보임

### 1-3. `/guide` 인덱스의 현재 모습

```
강의 가이드 (h1)
[원데이 클래스 4주차 시리즈] (1개 카드, 1주차만 active)
[폴더 구조 / 사이드바 내비 구조도]   ← 개발자용 정보, 일반 사용자에겐 노이즈
```

문제:
- 단순 안내 카드 1개 + 구조도. **커리큘럼 카탈로그 역할 못 함**
- Week1/2/3/4 4주차 모두 카드로 안 보임 (1주차만)
- 각 커리큘럼의 "곧 열릴 회차" 정보 0
- 구조도(폴더 트리)는 운영자 본인용으로는 OK인데 일반 사용자에겐 정보 노이즈

### 1-4. `/guide/oneday/week1` 의 현재 모습 (이건 잘 됐음)

```
1주차 교안 · CURRICULUM
"Claude로 첫 웹사이트 만들기 — 1주차"
[예정된 회차 위젯] ← 같은 curriculumId 의 events 자동 노출 (이미 잘 됨)
[OnedayClassCurriculum 본문]
```

**Week1 의 UpcomingClasses 위젯이 모범 사례** — 같은 패턴을 메인 + /guide 인덱스로 끌어올리면 됨.

  { "id": "oneday-week-1", "title": "...", "guideRoute": "/guide/oneday/week1", ... }

Week2/3/4 도 `oneday-week-2/3/4` 로 추가하고, `status: 'preparing'` 같은 필드로 "오픈 예정" 구분. 그러면 curriculums.json 단일 소스로 메인·/guide·Week*.jsx 모두 동기화.

## 3. 권장 정보 구조 (재정의)

| 페이지 | 핵심 가치 제안 | 보여주는 것 |
|--------|---------------|-----------|
| **`/` (메인)** | "지금 신청 가능한 회차 한눈에 + 다음 강의 미리보기" | 회차 카드(events) + 지도 + **커리큘럼 미리보기 4장 폴백** (회차 0건일 때) |
| **`/guide` (인덱스)** | "전체 강의 커리큘럼 카탈로그" | 커리큘럼 카드 4개 + 각 카드에 "다음 회차" 인라인 + "교안 보기" CTA |
| **`/guide/oneday/week{N}`** | "교안 본문" | 교안 본문 + 그 교안의 곧 열릴 회차 위젯 (Week1만 채워짐) |
| **`/events/:id`** | "이 회차 신청" | 회차 상세 + 신청 CTA |

**핵심 룰**:
- 메인 = **회차 중심** (오늘 신청 가능한 것 우선)
- /guide = **커리큘럼 중심** (장기 커리큘럼 안내)
- /events/:id = **신청 단위** (회차 1건)
- /guide/oneday/week{N} = **교안 단위** (커리큘럼 1건 본문)

별도 라우트 (`/curriculums`, `/classes`) 신설 불필요. 위 4개로 충분.

## 4. 구현 액션 (3단계)

### 4-1. curriculums.json 확장 (데이터 1차)

```json
[
  {
    "id": "oneday-week-1",
    "weekNumber": 1,
    "title": "1주차 — 미니홈피 만들기",
    "subtitle": "VSCode·Claude 설치부터 미니홈피 한 채 + 인터넷 공개까지",
    "summary": "비전공자도 두 시간 안에 첫 화면을 띄우고 진짜 URL 한 개를 가져갑니다.",
    "status": "active",                     // ← 새 필드: active / preparing
    "thumbnail": "/images/...",             // ← 새 필드: 카드 썸네일
    "guideRoute": "/guide/oneday/week1",
    "duration": "3시간", "level": "비전공자 첫 강의",
    "outline": [...], "prerequisites": [...], "outcomes": [...]
  },
  {
    "id": "oneday-week-2",
    "weekNumber": 2,
    "title": "2주차 — 프로필 + 방문자카운터",
    "subtitle": "내 소개 페이지에 프로필과 방문자카운터를 추가",
    "summary": "1주차 결과물에 localStorage 첫 도입.",
    "status": "preparing",
    "thumbnail": null,                      // 준비중이면 placeholder
    "guideRoute": "/guide/oneday/week2",
    "topics": [...]                         // Week2.jsx 안 TOPICS 외부화
  },
  // oneday-week-3, oneday-week-4 동일 패턴
]
```

이게 끝나면:
- Week2/3/4.jsx 가 curriculums.json 의 본인 메타 읽음 (제목·토픽 등 단일 소스)
- 메인·/guide 가 같은 데이터로 카드 렌더 가능

### 4-2. 컴포넌트 신규 — `CurriculumPreviewCard` + `CurriculumGrid`

| 컴포넌트 | 역할 | 위치 |
|---------|------|------|
| `CurriculumPreviewCard` | 커리큘럼 1개 카드 (썸네일·제목·다음 회차 인라인·CTA) | `front/src/components/CurriculumPreviewCard.jsx` |
| `CurriculumGrid` | 카드 N개 그리드 (active 우선 → preparing) | `front/src/components/CurriculumGrid.jsx` |

`CurriculumPreviewCard` props:
```jsx
<CurriculumPreviewCard
  curriculum={...}                  // curriculums.json 1건
  nextEvent={...}                   // 같은 curriculumId 의 곧 열릴 회차 1건 (없으면 null)
  variant="grid"                    // 'grid' | 'compact' (메인 폴백 vs /guide 카드)
/>
```

내부 로직:
- `curriculum.status === 'active' && nextEvent` → "다음 회차: 5/10(일) 14시 · 25,000원 · 잔여 4석" + "신청 페이지 →" 링크 (`/events/${nextEvent.id}`)
- `curriculum.status === 'active' && !nextEvent` → "다음 회차 일정은 카톡 오픈채팅에서 안내" + 카톡 CTA
- `curriculum.status === 'preparing'` → "오픈 예정" 배지 + "교안 미리보기 →" 링크 (`curriculum.guideRoute`)

### 4-3. 사용처 3곳 마운트

| 위치 | 무엇을 보여줄지 |
|------|---------------|
| **메인 — 카드 그리드 폴백** | `CurriculumGrid variant="grid"` (4장). filtered 가 0이거나 / 회차 1건이라 비어 보일 때 자연스럽게 채움 |
| **/guide 인덱스 — 메인 카탈로그** | `CurriculumGrid variant="grid"` (4장). 기존의 단조로운 SECTIONS 카드 1개를 대체. 구조도(폴더트리)는 admin 페이지로 이동 또는 제거 |
| **Week2/3/4.jsx — 본문 헤더** | `<CurriculumPreviewCard variant="compact" curriculum={...} />` 1장. 현재 인라인 TOPICS 와 통합 |

### 4-4. 메인 페이지 카드 그리드 폴백 동작 정의

```
[현재] events 0건 또는 1건뿐
  → "조건에 맞는 모임이 없어요" 텍스트 + admin "새 강의 추가" placeholder

[변경] events 게시된 게 있으면
  → 카드 그리드 (지금처럼)

[변경] events 게시된 게 0건 또는 추천 카드 1건이라 카드 그리드가 빈 경우
  → CurriculumGrid (4장 미리보기) + 그 위에 "📚 커리큘럼 미리보기" 헤더 1줄
  → admin은 그 옆에 "새 강의 추가" placeholder 유지
```

## 5. 라우팅 변경 — **신설 0개**

현재 라우트 그대로:
- `/` (메인)
- `/events/:id` (회차 상세)
- `/guide` (가이드 인덱스 — 역할 강화)
- `/guide/oneday/install`, `/guide/oneday/demo`
- `/guide/oneday/week1~4` (교안 본문)

신설하지 말 것:
- ❌ `/curriculums` — /guide 가 그 역할
- ❌ `/classes` — /events 가 그 역할 (필터링 view 라면 메인이 이미)
- ❌ `/series/:id` — 4주차 시리즈가 1개뿐인 지금은 과한 추상화

라우트 신설은 **"동선이 충돌해서 새 페이지가 필요한 시점"** 에만. 지금은 4페이지로 충분.

## 9. 결론 (TL;DR)

1. **별도 라우트 만들 필요 0**. 현재 4페이지(메인 / events:id / guide / guide/oneday/week:N)로 충분.
2. **/guide 인덱스를 "커리큘럼 카탈로그" 역할로 강화**. 지금은 너무 비어있음.
3. **메인 카드 그리드 빈 자리에 "교안 미리보기 4장" 폴백** (이미 합의된 A 안). 데이터 소스는 curriculums.json (4건으로 확장).
4. **curriculums.json 4건으로 확장 + Week2/3/4 가 단일 소스로** — 이게 빠지면 카드 4장 데이터 출처가 분산됨. **이게 가장 먼저 해야 할 작업**.
5. 컴포넌트 신규 2개 (`CurriculumPreviewCard`, `CurriculumGrid`) 면 끝. 메인·/guide·Week2/3/4 본문 모두 같은 컴포넌트로.

작업 시간 약 2시간. 위험도 low.
