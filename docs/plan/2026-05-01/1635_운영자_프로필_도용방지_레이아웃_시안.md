# 운영자 프로필 — 도용 방지 시그니처 분산 배치 시안

> 작성: 2026-05-01 16:35
> 대상: 홈 + 강의 페이지 + footer (시그니처 다중 배치)
> 목적: "내 사이트구나"라는 정체성을 여러 군데 박아 도용 진입장벽 형성

---

## 핵심 전략

> **한 군데에 크게 박지 말고, 5군데에 자연스럽게 박는다.**
> 도용자가 1군데 지우면 4군데가 남는다. 다 지우려면 사이트를 통째로 다시 디자인해야 함 → 그럴 바엔 안 베낌.

| # | 위치 | 강도 | 도용 시 부담 |
|---|------|------|-------------|
| ① | 홈 히어로 — "이 모임 운영자" 한 줄 소개 카드 | 중 | 메인 페이지라 안 지우면 본인 노출 |
| ② | `/about` 또는 홈 하단 — 운영자 풀 프로필 섹션 | 강 | 사진+이름+이력. 떼면 신뢰도 0 |
| ③ | 강의 페이지(1주차) 상단 — "강사 마이크로 카드" | 약 | 작지만 강의마다 박혀서 다 떼야 함 |
| ④ | Footer — 운영자 + 라이선스 (앞 시안에 포함) | 약 | 모든 페이지에 박힘 |
| ⑤ | 교안 코드 샘플 주석 — `// © 바이브 세션 / 2026` | 미세 | 코드 베껴 가면 같이 따라감 |

이번 시안에서는 **①②③** 만 그림. ④는 앞 시안(footer)에서 이미 다뤘고, ⑤는 코드 작성 시 자연스럽게 들어감.

## ① 홈 히어로 옆 — "운영자 한 줄 카드"

홈 첫 화면에 자체 강의 배너 옆 또는 아래에 작게.

### 모바일

```
┌──────────────────────────────────────┐
│   [히어로 이미지 / 자체 강의 배너]   │  ← 기존
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│  ●●  안녕하세요, 운영자 [이름]입니다 │  ← 14px font-semibold
│                                      │
│  10년차 개발자가 비전공자 친구들을   │  ← 13px slate-600
│  위해 만드는 작은 코딩 모임이에요.   │
│                                      │
│  → 더 알아보기                       │  ← 12px brand-600
└──────────────────────────────────────┘
   ↑ 좌측에 둥근 아바타(56px) 배치, 우측 텍스트
   ↑ rounded-2xl, border-slate-200, p-4, bg-white
```

### 데스크탑 (1024px+)

→ 히어로 우측에 사이드 카드로 붙거나, 히어로 아래 풀폭으로 1줄.

### Tailwind 스케치

```jsx
<section className="mt-6 rounded-2xl border border-slate-200 bg-white p-4 sm:p-5 flex gap-4 items-start">
  <img
    src="/images/operator-avatar.jpg"
    alt="운영자 [이름]"
    className="w-14 h-14 rounded-full object-cover shrink-0 ring-2 ring-brand-100"
  />
  <div className="min-w-0">
    <div className="text-[14px] font-semibold text-slate-800">
      안녕하세요, 운영자 <span className="text-brand-700">[이름]</span>입니다
    </div>
    <p className="mt-1 text-[13px] text-slate-600 leading-relaxed">
      10년차 개발자가 비전공자 친구들을 위해 만드는 작은 코딩 모임이에요.
    </p>
    <Link to="/about" className="mt-2 inline-block text-[12px] text-brand-600 hover:underline">
      → 더 알아보기
    </Link>
  </div>
</section>
```

## ② `/about` 또는 홈 하단 — 운영자 풀 프로필

본인 얼굴 + 이력 + 왜 이 모임을 만들었는가. **이게 가장 강력한 도용 방지**.
도용자가 이 섹션을 그대로 두면 본인 정체가 노출되고, 떼면 신뢰도 0인 사이트가 됨.

### 모바일

```
┌──────────────────────────────────────┐
│            ┌──────────┐              │
│            │          │              │  ← 정사각 사진(160px), 둥글게
│            │  사진    │              │     운영자 본인 사진
│            │          │              │
│            └──────────┘              │
│                                      │
│      [이름] · 10년차 개발자          │  ← 18px font-bold center
│                                      │
│   "코딩 처음 하시는 분들이 부담없이  │  ← 14px italic, slate-700
│    첫 화면을 띄워보는 경험을 만드는  │     운영자의 한 마디
│    게 좋아서 이 모임을 시작했어요."  │
│                                      │
│  ─────────────────────────────       │
│                                      │
│  이력                                │  ← 11px tracking-widest uppercase
│  · 前 [회사] 백엔드 엔지니어          │
│  · React/Node.js 사이드프로젝트 다수 │  ← 13px slate-700
│  · 2024년부터 비전공자 코딩 모임 운영│
│                                      │
│  ─────────────────────────────       │
│                                      │
│  연락                                │
│  📷 @instagram_handle                │
│  💬 카톡 오픈채팅                    │
│  ✉ email@domain.com                  │
└──────────────────────────────────────┘
```

### 데스크탑

│   ┌──────────┐    [이름] · 10년차 개발자                    │
│   │          │                                             │
│   │  사진    │    "코딩 처음 하시는 분들이..."              │
│   │          │                                             │

└────────────────────────────────────────────────────────────┘

> 위치 옵션 2가지 — 본인 선택:
> - **(A) 홈 페이지 하단에 풀폭 섹션으로 박기** — 메인 진입자 모두에게 노출. 추천.
> - **(B) `/about` 별도 페이지** — 깔끔하지만 클릭해야 봄. 노출도 낮음.
> → 1단계는 **(A) 홈 하단 + (B) /about에도 동일 콘텐츠 재사용** 둘 다 박는 게 안전.

## ③ 강의 페이지 상단 — "강사 마이크로 카드"

1주차 페이지 상단(히어로 바로 아래)에 작게.
"이 강의 만든 사람"을 명시 → 도용 시 강의별로 다 떼야 함.

### 모바일

```
[히어로 배너 — 기존]

┌──────────────────────────────────────┐
│ ●  강사 [이름]  · 운영자             │  ← 12px slate-600
│    "처음 코드를 쓰는 분 환영합니다"   │  ← 11px slate-500
└──────────────────────────────────────┘

[OnedayClassCurriculum 본문 — 기존]
```

### Tailwind 스케치

```jsx
<div className="mt-3 mb-4 flex items-center gap-2.5 text-[12px] text-slate-600">
  <img
    src="/images/operator-avatar.jpg"
    alt="강사 [이름]"
    className="w-7 h-7 rounded-full object-cover ring-1 ring-slate-200"
  />
  <span>
    <span className="font-semibold text-slate-800">강사 [이름]</span>
    <span className="text-slate-400"> · 운영자</span>
  </span>
  <span className="hidden sm:inline text-slate-400">·</span>
  <span className="hidden sm:inline text-slate-500 italic">
    "처음 코드를 쓰는 분 환영합니다"
  </span>
</div>
```

`EventDetail.jsx`의 히어로 배너와 카드 사이에 1줄 삽입. 자체 강의(`event.source === 'internal'`)일 때만 노출.

## 시그니처 분산 결과 — 도용자 입장 시뮬레이션

도용자가 이 사이트를 통째로 베낀다고 가정:

| 위치 | 도용자가 해야 할 일 |
|------|--------------------|
| ① 홈 히어로 카드 | 사진·이름·소개 다 바꿔야 함. 안 바꾸면 메인 페이지에 본인 정체 박힘 |
| ② /about 풀 프로필 | 사진·이력·인용·연락처 4개 다 바꿔야 함. 떼면 신뢰도 0 |
| ③ 강의 마이크로 카드 | 강의 N개마다 반복. 빠뜨리면 들킴 |
| ④ Footer | 모든 페이지마다 박혀있음. 1군데라도 빠뜨리면 들킴 |
| ⑤ 코드 주석 | 코드 베껴가면 자동으로 따라감. 일일이 지워야 함 |

→ **5군데를 다 정확히 갈아끼우는 작업량 > 처음부터 새로 만드는 작업량**.
→ 그럴 바엔 안 베낌. 도용 진입장벽 형성 완료.

## 추가 보강 — 미세한 시그니처 (선택)

진입장벽을 더 높이고 싶으면:

- **사이트 favicon** — 본인 이니셜이나 모임 로고. 도용자가 안 바꾸면 브라우저 탭에 박힘
- **OG 이미지 (카톡·인스타 공유 시 미리보기)** — 운영자 사진 + 사이트명. 공유될 때마다 따라감
- **404 페이지** — "없는 페이지에요. 운영자 [이름]이 만든 사이트로 돌아가시려면 [홈]" 같은 카피
- **메타 태그** — `<meta name="author" content="[이름]">`, `<meta name="copyright" content="© 2026 바이브 세션">` (눈에 안 보이지만 검색 엔진/크롤러가 인덱싱)

이건 도용 방지보다는 **법적 출처 입증** 용. 분쟁 시 "이게 내 사이트다"를 메타 태그·OG 이미지·favicon 으로 입증 가능.

## 결정 필요한 것 (시안 보고 답해줘)

1. **②의 위치** — (A) 홈 하단 풀폭 + /about 양쪽 다 박기 / (B) /about만 / (C) 홈 하단만. **(A) 추천.**
2. **사진 톤** — 정장 프로필 / 캐주얼 후드티 / 모임 진행 중 사진 / 이모지·일러스트 (얼굴 안 까고 싶을 때). 어떤 톤?
3. **이력 공개 수준** — 회사명 노출 OK? "前 [회사]" / "前 대기업 백엔드" / "10년차 개발자"만. 어디까지?
4. **운영자 본인 정보** — 이름 / 인스타 / 카톡 / 이메일 / 사진 — 알려주면 placeholder 다 채워서 코드까지 한 번에 진행. 시안만 확정하고 정보는 나중에 채울지.

이 4개만 답해주면 시안대로 컴포넌트 3개 만들어서 홈/강의/about 동시에 적용할게.
