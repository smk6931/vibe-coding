# Chapter 3 — Vercel 배포 가이드

미니홈피를 인터넷에 올려서 친구한테 링크 보내자.
GitHub에 올린 프로젝트를 **5분 만에 무료** 배포한다.

---

## 사전 준비

- Chapter 1에서 GitHub에 저장소 push 완료
- Chapter 2에서 미니홈피 작업한 거 다 commit + push 완료
- (확인) `https://github.com/내아이디/저장소이름` 들어가서 코드 보이면 OK

---

## Vercel 배포 5단계

### 1단계 — Vercel 가입

1. [https://vercel.com](https://vercel.com) 접속
2. 우측 상단 **"Sign Up"** 클릭
3. **"Continue with GitHub"** 선택
4. GitHub 권한 화면에서 **"Authorize Vercel"** 클릭

### 2단계 — 새 프로젝트 만들기

1. 대시보드에서 **"Add New..."** → **"Project"** 클릭
2. **"Import Git Repository"** 목록에서 내 저장소 찾기
3. 저장소 옆 **"Import"** 버튼 클릭

> **저장소가 안 보이면?**
> "Adjust GitHub App Permissions" 클릭 → 내 저장소를 Vercel이 볼 수 있게 권한 추가

### 3단계 — 배포 설정 (그냥 Deploy 누르면 됨)

Configure Project 화면에서 자동으로 잡히는 값 확인:

| 항목 | 자동 감지 값 |
|------|-------------|
| Framework Preset | Vite |
| Build Command | `npm run build` |
| Output Directory | `dist` |
| Install Command | `npm install` |

**아무것도 안 건드리고 그대로 "Deploy" 클릭.**

### 4단계 — 배포 대기

- 1~2분 빌드 진행
- 폭죽 화면 + "Congratulations!" 뜨면 성공
- **"Continue to Dashboard"** 클릭 → 내 사이트 URL 확인

### 5단계 — 사이트 접속

- URL 형식: `프로젝트이름-xxx.vercel.app`
- 클릭 또는 복사해서 새 탭에서 열기
- 미니홈피 화면 정상 출력 확인 → **배포 완료**

---

## 자동 재배포 확인 (Vercel의 진짜 매력)

이제 코드 고치고 push만 하면 자동으로 다시 배포된다.

1. VS Code에서 About 페이지 글자 한 줄 수정
2. Source Control → 커밋 → Sync (push)
3. 1~2분 후 사이트 새로고침 → 변경사항 반영됨
4. Vercel 대시보드 → "Deployments" 탭에서 새 배포 자동 생성된 거 확인

---

## 흔한 문제

### 흰 화면만 뜬다
- F12 → Console 탭에서 빨간 에러 확인
- 대부분 원인: `vite.config.js`의 `base` 설정 문제
- Vercel은 루트 배포라 `base: '/'` (기본값) 이어야 함

### 빌드 실패
- Vercel 대시보드에서 "View Build Logs" 클릭
- 빨간 줄 찾아서 AI한테 그대로 복사해서 물어보기:
  > "Vercel 빌드에서 이 에러 났어. 뭐가 문제야? [에러 복붙]"

### URL이 너무 길다
- Project Settings → Domains
- "Add" 클릭해서 `내이름-minihome.vercel.app` 같이 짧게 변경
