## 1. 가이드 페이지 레이아웃 (GuideLayout + GuideSidebar)

### 문제
데스크탑에서 사이드바가 너무 좁고, 콘텐츠 영역도 좁음. `flex`에 `maxWidth: 900px` 하드코딩.
`lg:` (1024px)로 사이드바 분기했더니 940px 뷰포트에서 사이드바가 상단 드롭다운으로 나와 불편.

### 왜 이렇게 고쳤나
- `flex` → **CSS Grid** (`gridTemplateColumns: '200px 1fr'`)로 교체
  사이드바 고정 200px + 콘텐츠 나머지 전부 — max-width 제한 없앰
- `lg:` → **`md:`(768px)** 기준으로 사이드바 표시/숨김
  태블릿(768-1023px)에서도 사이드바가 왼쪽에 붙어야 자연스러움
- `md:sticky md:top-24`: 사이드바 스크롤 시 고정
- `min-w-0` on main: grid cell overflow 방지

### 변경 파일
- `front/src/pages/guide/GuideLayout.jsx` (전면 재작성)
- `front/src/pages/guide/GuideSidebar.jsx` (전면 재작성, 모바일 드롭다운 추가)

## 3. MiniHompy 데모 컴포넌트 신규 제작

### 결정: 파일 위치를 `pages/guide/oneday/`에 co-locate

처음에 `components/`에 만들었다가 이동. 이유:
- 이 컴포넌트는 가이드 설치 페이지의 "AI 결과물 시연" 전용
- `components/`는 2곳+ 재사용 컴포넌트용
- 단일 사용처 컴포넌트는 사용처 폴더에 놓는 것이 명시적

### 구현 시 핵심 이슈

**캔버스 범위**: 원본 test 프로젝트는 `window.innerWidth/Height` 기준 → 화면 전체에 별이 깔림.
→ `canvas.parentElement.offsetWidth/Height` 기준으로 수정, `position: fixed` → `position: absolute`로.

**CSS 충돌**: 전역 CSS 없이 **CSS Modules** 사용. `@keyframes` 이름에 `mh` 접두사 필수.

**비주얼**: `max-width: 660px`, `::before/::after` 글로우 오브, shimmer 헤더, 4열 favorites.

### 변경 파일
- `front/src/pages/guide/oneday/MiniHompy.jsx` (신규)
- `front/src/pages/guide/oneday/MiniHompy.module.css` (신규)
- `front/index.html` (Google Fonts 추가: Cute Font, Nanum Gothic Coding, Noto Sans KR)
