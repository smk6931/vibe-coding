# 레이아웃 개선 + 모바일 우선 룰 + 카카오맵 통합

작성: 2026-04-25 12:30
작업자: 운영자 + Claude
선행 문서: `1145_프로토타입_데모_초기구축.md`

## 작업 배경

사용자 피드백 4가지:
1. 데스크탑(초광폭 모니터)에서 컨테이너가 가운데 좁게 떠서 황량함 — 레이아웃이 구리다.
2. 사이드바를 상단에 둘지 좌측에 둘지 결정 필요.
3. 모바일 위주로 가는 게 맞음 (인스타·당근·카톡 공유 → 휴대폰 진입이 다수).
4. SVG 임시 지도 말고 실제 지도(카카오맵 등) 쓰고 싶음.

## 왜 이 방법을 골랐나

### 1. 사이드바 = 상단 헤더 + 모바일 햄버거
- 외부 노출 페이지(랜딩/대시보드/라이브러리/커뮤니티)는 가로 폭이 핵심(특히 지도 + 캘린더). 좌측 사이드바를 두면 200~240px가 영구 손실.
- 모바일 우선 정책에서는 좌측 사이드바보다 햄버거가 표준.
- 어드민·마이는 추후 페이지 내부 좌측 sub-nav 허용 (현재는 단일 페이지라 불필요).

### 2. 카카오맵 vs 네이버맵 vs Leaflet/OSM
- 카카오: 일 30만 호출 무료, 한국 POI 정확, 가입 + JavaScript 키 1개. **선택**.
- 네이버: 일 10만, 가입 절차 더 복잡.
- Leaflet/OSM: 키 불필요하지만 한국 POI(스터디룸 등) 빈약 → 운영자가 주소·이름 표기를 손으로 하게 됨.
- **결정**: 카카오맵. `react-kakao-maps-sdk` 의 `useKakaoLoader` + `<Map>` + `<CustomOverlayMap>` 패턴.

### 3. 키 미설정 시 SVG fallback 유지
- **이유**: 키 발급 전에도 데모/배포가 동작해야 함. 사용자가 키 등록을 미루더라도 사이트 전체가 멈추면 안 됨.
- **구현**: `process.env.NEXT_PUBLIC_KAKAO_MAP_KEY` 존재 여부로 분기. 키 있으면 `KakaoMultiMap`/`KakaoSingleMap`, 없으면 기존 SVG 컴포넌트.
- 키 등록 후 dev 재시작이면 자동 전환.

### 4. CLAUDE.md를 프로젝트 루트에
- 글로벌 CLAUDE.md는 "UI/배포는 프로젝트 로컬에" 라고 명시 분리.
- 따라서 `c:\GitHub\vibe_coding\CLAUDE.md` 신규. 디자인 톤·지도·결제·배포 룰 모두 여기.
- 향후 `backend/` 추가 시에도 같은 루트 룰 공유.

### 5. 컨테이너 폭 max-w-7xl + container-wide(1440)
- 기존 max-w-6xl(1152px)이 초광폭 모니터에서 작게 떠 보였음.
- 새 정책: 일반 페이지 `max-w-7xl`, 대시보드처럼 가로 폭이 핵심인 페이지는 `max-w-[1440px]` 의 `.container-wide`.
- 모바일 padding은 `px-4`, 데스크탑 `sm:px-6 lg:px-8`.
