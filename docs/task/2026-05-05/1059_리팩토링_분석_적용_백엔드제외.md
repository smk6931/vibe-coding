# 리팩토링 분석 결과 적용 (백엔드 섹션 제외)

선행 문서: `docs/plan/2026-05-05/0946_프로젝트_리팩토링_분석.md`

## 배경

방금 작성한 리팩토링 분석 보고서의 항목 중 백엔드(섹션 7)를 제외한 모든 즉시·다음 스프린트 항목을 한 번에 적용. 장기 항목(HomeClient/GuideClient 책임 분리, ChapterGuide 통합)은 위험도 med + 작업 흐름 영향이 커서 별도 결정 사안으로 보류.

## 왜 한 번에 적용했나

- 보고서의 **즉시 5건 + 다음 4건** 은 모두 위험도 low 또는 1줄~10줄 수준 변경. 묶어서 처리해도 PR 추적 어렵지 않음.
- 장기 항목(책임 분리, 컴포넌트 통합)은 의도적 설계 결정이 끼어 있어 사용자 합의 없이 바로 가는 게 부담.
- 결정 필요 항목 A~H 중 안전한 기본값(B 옵션 — 문서에서 제거, /library → /guide 리다이렉트)으로 처리하고 장기 결정은 보고서에 그대로 남김.
