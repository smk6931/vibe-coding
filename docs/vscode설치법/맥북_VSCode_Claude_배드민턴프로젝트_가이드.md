# 맥북 완전 초보자를 위한 VSCode + Claude 설치 & 배드민턴 대시보드 프로젝트 가이드

> 맥북을 처음 쓰는 분도 걱정 없습니다.  
> Windows와 다른 부분만 짚어드리고, 모르는 건 전부 Claude한테 물어보면 됩니다.

---

## 맥북 vs Windows — 핵심 차이점만 먼저 알기

| 기능 | Windows | 맥북 |
|---|---|---|
| 복사 | Ctrl+C | **⌘(Command)+C** |
| 붙여넣기 | Ctrl+V | **⌘+V** |
| 저장 | Ctrl+S | **⌘+S** |
| 전체 선택 | Ctrl+A | **⌘+A** |
| 화면 캡처 | Win+Shift+S | **⌘+Shift+4** (드래그 선택) |
| 파일 탐색기 | Windows 탐색기 | **Finder** (독 맨 왼쪽 웃는 얼굴) |
| 터미널 | VSCode 내 터미널 | **VSCode 내 터미널** (동일) |
| 단축키 Ctrl 역할 | Ctrl | 대부분 **⌘(Command)** 로 대체 |

> **⌘ 키** = 키보드 스페이스바 왼쪽에 있는 cmd 또는 ⌘ 표시 키

## PART 1 — 맥북 설치 (3단계)

---

### 1단계 — VSCode 설치

1. 브라우저에서 **https://code.visualstudio.com** 접속
2. **Download for Mac** 버튼 클릭 (Apple Silicon이면 "Apple Silicon", 구형이면 "Intel" 선택)
   - 모르면 그냥 큰 파란 버튼 클릭 (자동 감지됨)
3. 다운로드된 `.zip` 파일이 **다운로드 폴더**에 생성됨
4. 파일 더블클릭 → `Visual Studio Code.app` 파일이 나타남
5. `Visual Studio Code.app`을 **응용 프로그램(Applications)** 폴더로 드래그
6. 독(화면 하단 바)에서 VSCode 아이콘 클릭해서 실행

> **주의**: 처음 실행하면 "인터넷에서 다운로드한 앱입니다" 경고창이 나올 수 있습니다.  
> **열기** 버튼 클릭하면 됩니다. 안전한 프로그램입니다.

---

### 2단계 — Node.js 설치

1. 브라우저에서 **https://nodejs.org/ko/download** 접속
2. **LTS 버전** 다운로드 클릭
   - macOS용 `.pkg` 파일 다운로드
3. 다운로드된 `.pkg` 파일 더블클릭
4. 설치 마법사에서 **계속 → 계속 → 설치** 클릭
5. 비밀번호 입력 요청하면 **맥북 로그인 비밀번호** 입력

### 설치 확인

VSCode 실행 → 상단 메뉴 **터미널 → 새 터미널** → 아래 입력:

```
node --version
```

`v22.x.x` 같은 숫자가 나오면 성공

---

### 3단계 — Claude Extension 설치 및 로그인

1. VSCode 왼쪽 사이드바 **Extensions 아이콘** 클릭
   - 네모 4개 겹친 모양 (단축키: **⌘+Shift+X**)
2. 검색창에 **claude** 입력
3. **Claude for VS Code** — Anthropic 공식 선택 → **Install**
4. 설치 후 왼쪽 사이드바 **Claude 아이콘** 클릭
5. **Sign In** → 브라우저에서 Anthropic 계정 로그인
   - 계정 없으면 **https://claude.ai** 에서 회원가입 먼저

## PART 2 — 배드민턴 대시보드 프로젝트 계획

---

### 프로젝트 개요

**목표:** 배드민턴 선수들의 경기 영상 분석 데이터를 기반으로  
풋볼 매니저 게임처럼 선수 컨디션, 전략, 통계를 한눈에 볼 수 있는  
웹 대시보드를 만든다.

---

### 단계별 로드맵

```
Phase 1 (1~2주)  →  기본 대시보드 화면만 만들기 (데이터 없이 디자인만)
Phase 2 (3~4주)  →  선수 데이터 입력 기능 추가
Phase 3 (5~6주)  →  영상 분석 데이터 연동
Phase 4 (7주~)   →  고급 기능 (AI 분석, 전략 추천 등)
```

> **처음부터 전부 다 만들려고 하지 말 것.**  
> Phase 1 화면만 완성해도 엄청난 성과입니다.

---

### Phase 1 — 화면 레이아웃 설계

아래 화면들을 하나씩 만들어 나갑니다.

#### 메인 대시보드 (홈 화면)
- 팀 전체 현황 카드 (승/패, 이번 시즌 성적)
- 선수 컨디션 요약 (신호등 표시: 좋음/보통/주의)
- 다음 경기 일정

#### 선수 상세 페이지
- 선수 프로필 (사진, 이름, 포지션)
- 컨디션 그래프 (주간/월간)
- 최근 경기 성적
- 강점/약점 레이더 차트

#### 경기 분석 페이지
- 경기별 통계 (스매시 성공률, 서브 실수, 랠리 길이 등)
- 전략 메모 기록
- 상대팀 패턴 분석

---

### Phase 1 시작 프롬프트 (이것만 복사해서 Claude에 붙여넣기)

```
나는 코딩을 전혀 모르는 완전 초보자야.
맥북을 쓰고 있고 VSCode + Claude 환경이 세팅되어 있어.

배드민턴 팀 관리 웹 대시보드를 만들고 싶어.
풋볼 매니저 게임처럼 선수 컨디션, 경기 통계, 전략을 한눈에 볼 수 있는 사이트야.

지금은 1단계로 화면 레이아웃만 먼저 만들고 싶어. 실제 데이터는 나중에 연결할게.

React로 아래 3개 화면을 만들어줘:
1. 메인 대시보드: 팀 성적 요약, 선수 컨디션 신호등, 다음 경기 일정
2. 선수 프로필: 사진, 이름, 컨디션 그래프, 강점/약점 레이더 차트
3. 경기 분석: 경기별 통계, 전략 메모

디자인은 어두운 배경(다크모드)에 깔끔하고 전문적인 느낌으로 해줘.
단계마다 터미널에 붙여넣을 명령어랑 다음에 뭘 해야 하는지 알려줘.
모르는 용어는 쉽게 설명해줘.
```

---

### 어떤 데이터를 넣을 수 있나요?

**선수 데이터 (Phase 2에서 입력 가능)**
- 이름, 나이, 포지션 (단식/복식/혼합복식)
- 체중, 신장, 소속팀
- 최근 훈련 강도 (1~10점)
- 수면 시간, 부상 여부

**경기 통계 (Phase 3에서 연동)**
- 스매시 성공률
- 서브 실수 횟수
- 랠리 평균 길이
- 셔틀콕 위치 히트맵
- 상대방 약점 공략 성공률

**영상 분석 데이터 (Phase 3~4)**
- 영상 타임스탬프별 메모
- 패턴 분류 (백핸드 약점, 네트 앞 처리 등)
- AI 자동 분석 (선택 사항)

---

### 막혔을 때 쓰는 프롬프트 모음

**설치/실행이 안 될 때**
```
맥북에서 [상황 설명]이 안 되고 있어.
터미널에 [오류 메시지 붙여넣기] 이런 오류가 나.
맥북 기준으로 해결 방법 알려줘.
```

**화면에 표시하고 싶은 게 생겼을 때**
```
지금 대시보드에 [추가하고 싶은 것]을 추가하고 싶어.
어느 파일을 수정해야 하는지, 전체 코드를 보여줘.
```

**디자인을 바꾸고 싶을 때**
```
[특정 화면]의 디자인을 바꾸고 싶어.
[원하는 느낌 설명] 스타일로 수정된 전체 코드 보여줘.
```

**경기 분석 기능을 추가하고 싶을 때**
```
경기 영상을 보면서 타임스탬프마다 메모를 남길 수 있는 기능을 추가하고 싶어.
어떻게 구현할 수 있는지 설명해주고 코드도 보여줘.
```

## PART 3 — 맥북 자주 발생하는 문제

| 문제 | 해결 방법 |
|---|---|
| "개발자를 확인할 수 없습니다" 경고 | 시스템 설정 → 개인정보 보호 및 보안 → 무시하고 열기 |
| 터미널에서 명령어가 안 먹힘 | 터미널 창 클릭 후 다시 입력, 또는 VSCode 재시작 |
| Claude 아이콘이 안 보임 | VSCode 완전 종료(⌘+Q) 후 재시작 |
| 한글 입력 안 됨 | VSCode 재시작 또는 ⌘+Shift+P → "reload window" |
| node 명령어가 없다고 나올 때 | Node.js 설치 후 터미널 완전히 닫고 다시 열기 |
| 화면 캡처가 저장만 되고 붙여넣기 안 됨 | ⌘+Shift+4 → 촬영 직후 ⌘+V (클립보드에 저장됨) |

## 전체 진행 흐름 요약

```
① 맥북에 VSCode + Node.js + Claude 설치 완료
        ↓
② VSCode 열기 → Claude 패널 열기
        ↓
③ Phase 1 시작 프롬프트를 Claude에 붙여넣기
        ↓
④ Claude가 알려주는 터미널 명령어 실행
        ↓
⑤ 코드 파일 교체 → ⌘+S 저장
        ↓
⑥ 브라우저에서 화면 확인
        ↓
⑦ 마음에 안 들면 "이 부분 바꿔줘" 요청
        ↓
⑧ Phase 2, 3로 단계적으로 기능 추가
```

## 꼭 기억할 것 4가지

| 번호 | 규칙 |
|---|---|
| 1 | 맥북에서 저장은 **⌘+S** (Ctrl 아님) |
| 2 | 오류나면 터미널 빨간 글씨 전체를 **⌘+C → Claude에 ⌘+V** |
| 3 | 한 번에 한 가지씩만 요청 (한꺼번에 많이 바꾸면 오류 추적이 어려움) |
| 4 | 막히면 화면 캡처(**⌘+Shift+4**) 후 Claude에 붙여넣기 |
